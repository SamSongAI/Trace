Resources

把更耐久的 Agent 理解，沉淀成可以回访的主题页。

Resources 承担的是“站点长期记忆”角色。它不追逐时间线，而是组织概念、范式、系统设计和阅读路径，让新内容有地方被吸收，不只是被发布。

Tracks

优先展开的主题方向

这些不是最终分类树，而是第一批最有必要建立稳定入口的主题带。后续收到的文章会优先被归入这些方向。

Agent Paradigms

单 Agent、多 Agent、反思循环、计划执行、长期任务结构等核心范式。

Orchestration

工作流编排、节点拓扑、状态传递、失败恢复与多步骤协作设计。

Memory & Context

上下文压缩、检索增强、会话记忆、长期记忆与环境状态建模。

Tool Use

工具调用、协议层、动作接口、权限边界和执行稳定性。

Evaluation

Agent 评测、失败分类、稳定性验证、任务成功率与可观测性。

Productization

从研究叙事到真实产品体验、前端结构、运维约束与商业化路径。

Formats

Resources 里的内容形态

不是每条内容都该成为博客文章。更适合长期使用的内容，会被整理成更稳定的资源形态。

Topic Page

围绕一个概念或主题建立背景、关键问题、代表性来源与站内判断。

Reading Map

把分散的文章、论文、发布说明串成一条可跟进的学习路径。

Pattern Note

把工程实践里的高频模式、坑点与 design trade-offs 归档成短结构页。

Status

当前状态

Resources 入口已经建立，但首批页面还在编排中。后续你发来的文章会先经过筛选，再决定进入这里还是进入 Blog。

适合进入 Resources 的内容

更耐久、解释性更强、适合被反复引用和链接的内容。

不适合直接进入 Resources 的内容

纯时间性更新、单点新闻、短期热议但没有沉淀价值的文章。

Resources are the long-term memory layer of SOTA SYNC.
