Read Me

这是一个一人公司的官网，也是一个内容与产品共用的出版入口。

SOTA SYNC 的站点不是把“公司介绍”和“内容策展”硬拆成两个世界，而是让观点、产品和工作流证明彼此。内容告诉你我在追踪什么，产品告诉你这些判断如何真的被用起来。

Why

为什么要做 SOTA SYNC

因为 Agent 领域的高质量讨论正在快速堆积，但真正值得回访、能帮助形成工作方法的内容仍然分散。SOTA SYNC 想做的是把这些高信号内容同步成一个更耐久、更有判断力的站点结构。

不是资讯站

重点不是更快，而是更清楚；不是多发，而是筛掉噪音后留下更高质量的表达。

不是纯产品官网

产品只是品牌表达的一部分。网站需要承载判断、策展和长期内容积累。

How

内容和产品如何连接

这里的内容不是独立于产品之外的观点展示。`Trace` 就是这种方法论的一次产品化：先留下结构化痕迹，再让后续系统和 Agent 接手。

Content Layer

Resources 和 Blog 负责解释前沿概念、整理范式、记录变化与形成判断。

Product Layer

Trace 把“高频留痕、低摩擦输入、后续路由”变成一个真实可用的软件入口。

Operating Mode

这个站会怎么被运营

默认是静态优先、只读、非互动的网站。不会做评论区，也不会把内容中心变成信息流。所有新增文章都会先判断是否值得收录、该进入 Resources 还是 Blog，以及是否需要首页推荐。

Curate

先筛选，只让高信号内容进入站点。

Synthesize

把单点内容变成可理解、可关联、可沉淀的表达。

Ship

在轻量静态架构上稳定发布，兼顾低带宽和长期维护成本。

SOTA SYNC is a one-person company site built around durable judgment.
