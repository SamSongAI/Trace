# Markdown Native Card Note Examples

> 目标：只用 Markdown 原生语法构建“卡片笔记感”，不依赖插件。

## 1) 代码块卡片（稳定、边界清晰）

```text
问题：追加逻辑偶发错位
假设：容器识别以“最后一个块”为准导致偏移
行动：改为“最新一条”优先，并补回归测试
2026-03-03 23:40
```

---

## 2) 引用卡片（更轻量）

> 观察：用户更希望“可读性”优先，而不是花哨样式。
>
> 决策：时间戳保持纯文本，不加 CSS。
>
> 2026-03-03 23:41

---

## 3) 任务型卡片（任务列表）

- [ ] 统一快捷键录制交互
- [ ] 增加分割线配置
- [ ] 回归测试覆盖追加逻辑
- [x] 修复 callout 追加错位

---

## 4) 事实来源卡片（脚注）

结论：追加策略必须基于“最新一条可识别容器”而不是“最后一条”。[^append]

[^append]: 在本项目中，新笔记插入到分区顶部，因此“最新一条”位于最前面。

---

## 5) 双栏感（表格）

| 字段 | 内容 |
| --- | --- |
| 场景 | Daily 快速捕捉 |
| 格式 | 代码块 / 引用 |
| 时间戳 | 纯文本默认格式 |
| 分割 | `---` 或 `***` |

---

## 6) 引用 + 分割（推荐的轻量卡片模板）

> {{内容}}
>
> {{yyyy-MM-dd HH:mm}}
