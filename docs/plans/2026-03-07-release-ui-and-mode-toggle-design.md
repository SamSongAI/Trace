# FlashNote 发布版 UI 与写入目标切换设计

日期：2026-03-07

## 目标

本轮聚焦两件发版前必须收敛的表面能力：

1. 给捕捉面板提供一个无需打开设置的固定快捷操作，用于在 `条目（Daily）` 和 `文档（Inbox）` 两种写入目标之间一键切换。
2. 把桌面端 `Capture / Settings` 的视觉语言拉齐到官网，避免产品站和本地客户端出现两套风格。

## 交互决策

- 固定快捷键采用 `Shift + Tab`。
- 行为定义：仅在捕捉面板激活时生效，触发后立即切换 `noteWriteMode`。
- `Shift + Tab` 不开放为可录制快捷键，防止用户覆盖写入目标切换。
- 模式提示需要同时出现在：
  - 捕捉面板顶部当前目标胶囊
  - 捕捉面板底部目标状态栏
  - 设置页“写入流程”卡片

## UI 决策

- 桌面端配色切换到与官网一致的基底：
  - 深海军蓝背景
  - 橙色强调色
  - 高对比浅色内容卡片
- 设置页增加 Hero 区与指标卡，减少传统表单页的“系统偏默认”观感。
- 捕捉面板保持高密度，但强化：
  - 当前模式可见性
  - 快捷设置的结构化分组
  - 底部条目 / 文档两种目标的差异化状态

## 实现范围

- `AppSettings.NoteWriteMode` 增加图标、说明文案与 `toggled` 状态。
- `CapturePanelController` 拦截 `⇧Tab`，切换写入目标并将焦点送回输入区。
- `CaptureView` 与 `SettingsView` 共用新的 `FlashNoteTheme`。
- `BrandAssets` 文案切换到 `FlashNote / Markdown input layer`，与官网命名对齐。
- 为 `⇧Tab` 增加快捷键显示测试。

## 验收标准

- 面板内按 `⇧Tab` 能立即在条目 / 文档间切换。
- 切换后焦点仍回到正文输入区。
- 设置页明确展示固定切换方式，且快捷键录制不能覆盖它。
- `swift test` 全绿。
